# 개인정보 처리방침 (Privacy Policy)

**앱 이름:** 당신의 타임라인 (your-timeline) — Jira 프로젝트용 타임라인 Forge 앱
**문의:** cukirang@gmail.com
**최종 수정일:** 2026-06-03

본 앱은 Atlassian Forge 플랫폼 위에서 동작하며, 사용자의 Jira 데이터를 **표시·편집**하기 위한
프로젝트 타임라인(간트) 기능을 제공합니다. 본 방침은 본 앱이 어떤 데이터를 다루는지 설명합니다.

---

## 1. 앱이 저장하는 데이터

본 앱은 Atlassian Forge Storage(설치된 Jira 사이트 내부 저장소)에 **다음 항목만** 저장합니다.

| 저장 항목 | 내용 | 개인정보 포함 |
|----------|------|--------------|
| 수동 추가 공휴일 | 날짜(YYYY-MM-DD)와 휴일명 | 아니오 |
| 페이지 제목 | 타임라인 페이지에 표시할 제목 텍스트 | 아니오 |
| 이슈 색상 | 이슈 키(예: `MA-123`)별 사용자 지정 색상 코드 | 아니오 |

위 데이터에는 **이름·이메일·계정 ID 등 개인 식별 정보가 포함되지 않습니다.**
저장 데이터는 해당 Jira 사이트의 Forge Storage에만 보관되며, 앱 제거 시 함께 삭제됩니다.

## 2. 앱이 접근하지만 저장하지 않는 데이터

다음 데이터는 화면 표시 목적으로 **조회만** 하며, 앱 저장소에 저장하거나 외부로 전송하지 않습니다.

- 이슈 정보(요약, 시작일/마감일, 상태, 담당자, 상위 항목 등) — JQL 조회 결과를 화면에 렌더링
- 프로젝트의 배정 가능 사용자 목록(담당자 필터용) — 표시 후 메모리에서만 사용
- 사용자가 접근 가능한 저장된 필터 목록

모든 Jira 접근은 **현재 로그인한 사용자의 권한(as user)** 으로 수행되며, 사용자가 원래 볼 수 있는
데이터 범위를 벗어나지 않습니다.

## 3. 데이터 쓰기(편집) 동작

사용자의 명시적 조작 시 다음 작업을 Jira에 직접 반영합니다(앱이 별도로 보관하지 않음).

- **막대 드래그**: 해당 이슈의 마감일/시작일 필드 수정
- **필터 저장**: 사용자가 입력한 JQL을 Jira 저장 필터로 생성

## 4. 외부 통신 (Egress)

- **date.nager.at** (Nager.Date 공개 공휴일 API): 한국 공휴일 정보를 **연도 단위로 조회**합니다.
  요청에는 연도와 국가 코드(KR)만 전송되며, **개인정보나 Jira 데이터는 일절 전송하지 않습니다.**
  API 키도 사용하지 않습니다.

그 외 어떤 외부 서비스로도 데이터를 전송하지 않으며, 분석/광고/추적 도구를 사용하지 않습니다.

## 5. 개인정보 제3자 제공 / 판매

본 앱은 어떤 데이터도 제3자에게 제공하거나 판매하지 않습니다.

## 6. 데이터 보관 및 삭제

- 저장 데이터(1항)는 설치된 Jira 사이트의 Forge Storage에 보관됩니다.
- 앱을 제거(uninstall)하면 해당 저장 데이터는 삭제됩니다.
- 조회성 데이터(2항)는 영구 보관하지 않으며 화면 렌더링 후 폐기됩니다.
